\section{Introduction}

Large Language Models (LLMs) perform well on many tasks, but their internal
behavior is still hard to understand. This is a serious issue because LLMs can
hallucinate. A hallucination occurs when the model generates false or
unsupported information. Detecting these errors is important, especially in
high-stakes settings.

Many hallucination detection methods use external tools. For example, they may
ask another LLM to judge the answer, sample many responses, or verify claims
with retrieval. These methods can work well, but they are often slow and
expensive. Mechanistic methods offer a different solution. Instead of using
extra LLM calls, they inspect the hidden states of the same model that
generated the answer.

A common mechanistic approach is probing. A probe is a small classifier
trained on hidden states. It tries to predict whether the answer is correct or
hallucinated. Probing is useful because it can be fast and only needs one
forward pass. However, many probes look at only one layer--token position.
This is limiting. Uncertainty signals may not appear in the same place for
every answer. They can move across layers, tokens, tasks, and models.
Therefore, looking at only a few hidden states may miss important signals.

Activation-tensor methods address this problem by using more of the hidden
states. For a generated answer, we can represent the hidden states as a
tensor $A \in \mathbb{R}^{L \times T \times D}$. Here, $L$ is the number of
layers, $T$ is the number of generated tokens, and $D$ is the hidden
dimension. This tensor can be viewed like an image. The layer and token axes
form a 2D plane, and the hidden dimension acts like channels. ACT-ViT uses
this idea to detect hallucinations from the full activation tensor.

However, sequence-level detection is not enough for long-form answers. A long
answer may contain many correct parts and only a few uncertain parts. A model
that gives one score for the whole answer can say that the answer is likely
hallucinated. But it does not say where the suspicious regions are. This
makes the prediction harder to interpret and harder to use for correction or
verification.

Localizing uncertainty is difficult because most datasets do not provide
token-level labels. They only label the full response as correct or
incorrect. One possible approach is to extract atomic claims, verify each
claim, and label the tokens that belong to false claims. But this has a
limitation. The hidden-state signals that lead to a hallucination may appear
before the false claim is generated. They may also be spread across earlier
context. As a result, claim spans may not fully capture the true uncertainty
signals in the activations.

We address this problem using weakly supervised temporal localization
\citep{pilhyeonlee2021}. This idea is common in video understanding. In video action localization, the
model may only know that an action occurs somewhere in a video. It does not
know the exact frames where the action starts and ends. The model learns to
find the important frames using only the video-level label.

We apply the same idea to LLM hidden states. Instead of localizing an action
in video frames, we localize uncertainty in generated-token windows. Each
token is treated as a time step. For each token window, we collect the hidden
states across all layers. The goal is to identify which windows contain
activation patterns that indicate uncertainty.

Our method has two stages. In the first stage, we train an ACT-ViT encoder on
full activation tensors. This teaches the encoder to recognize hidden-state
patterns related to uncertainty. In the second stage, we freeze the encoder.
We then apply it to short token windows. This produces one feature vector per
window. A Mamba-based temporal head \citep{albertgu2024} processes these
window features and predicts one uncertainty logit per window.

Because we only have sequence-level labels, we train the second stage with
top-$k$ multiple-instance learning. The model selects the top-$k$ most
uncertain windows. It averages their logits to produce one sequence-level
logit. This sequence score is trained against the full-response label. In
this way, the model learns which regions are most responsible for the
uncertainty prediction without needing token-level labels.

This two-stage design makes the task easier. If the model had to learn both
uncertainty features and localization at the same time, training could be
noisy. Instead, the first stage learns useful activation features. The
second stage focuses on temporal aggregation and localization. Mamba is used
because it can process long sequences efficiently, making the method better
suited for real-time generation.

\paragraph{Contributions.}
\begin{enumerate}
    \item We frame activation-based hallucination detection as a weakly
    supervised temporal localization problem.
    \item We propose a two-stage model that combines an ACT-ViT activation
    encoder with a Mamba temporal head.
    \item We use top-$k$ multiple-instance learning to train localization from
    only sequence-level labels.
    \item We extend activation-tensor methods from whole-answer detection to
    localized uncertainty estimation for long-form LLM outputs.
\end{enumerate}
